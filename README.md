# OCR Studio (Image + Live Camera OCR)

An end-to-end OCR project built around **Qwen2-VL-2B-Instruct** for extracting text from:

- Single images
- Batch image uploads
- Live camera frames (via WebSocket)

The app uses a FastAPI backend, OpenCV-based preprocessing, and a browser UI served from the API.

## Project Structure

- `Image_and_Live_Camera_OCR.ipynb` – main notebook that sets up everything
- `ocr_engine.py` – OCR pipeline (generated by notebook)
- `main.py` – FastAPI server + OCR endpoints (generated by notebook)
- `static/index.html` – frontend UI (generated by notebook)

> `ocr_engine.py`, `main.py`, and `static/index.html` are created when you run the notebook cells.

## Features

- **Image OCR**
  - Single image mode
  - Batch mode for multiple files
  - Optional auto-crop for document region extraction
- **Live Camera OCR**
  - Sends frames periodically to backend
  - Returns real-time extracted text
  - Handles busy server state to avoid stacked inference calls
- **OCR Quality Pipeline**
  - Upscaling, shadow removal, denoising, sharpening, auto-rotation
  - Multiple OCR prompts with scoring to choose the best output (image mode)
  - Fast single-prompt OCR path for live camera frames
- **Health & Device Info**
  - `/health` endpoint reports runtime/GPU status

## Tech Stack

- Python
- FastAPI + Uvicorn
- OpenCV, NumPy, Pillow
- Transformers + Torch
- Qwen2-VL-2B-Instruct
- HTML/CSS/JavaScript frontend

## Setup

This notebook is primarily designed for a **Linux runtime (for example: Google Colab with GPU)**.

### 1) Open and run notebook
Run cells in order in `Image_and_Live_Camera_OCR.ipynb`:

1. Install dependencies
2. Generate `ocr_engine.py`
3. Generate `main.py`
4. Generate `static/index.html`
5. Start server + Cloudflare tunnel (optional public URL)

### 2) GPU recommendation
For practical speed, use a GPU runtime.

- Without GPU: inference can be very slow.
- With GPU (recommended): significantly faster OCR performance.

## Running Locally (without notebook generation)

If you already have generated files (`ocr_engine.py`, `main.py`, `static/index.html`), run:

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

Then open:

- `http://localhost:8000` for UI
- `http://localhost:8000/health` for health check

## API Endpoints

- `GET /` – serves OCR web UI
- `GET /health` – API/runtime health + device info
- `POST /ocr/image` – OCR for one uploaded image
  - form fields:
    - `file`: image file
    - `auto_crop`: `true|false`
- `POST /ocr/batch` – OCR for multiple uploaded images
  - form fields:
    - `files`: multiple image files
    - `auto_crop`: `true|false`
- `WS /ocr/camera` – live camera OCR endpoint (base64 image frames)

## Notes

- The model used is `Qwen/Qwen2-VL-2B-Instruct` from Hugging Face.
- First run may take time due to model download/loading.
- Cell 5 includes a Cloudflare tunnel helper script using Linux paths (`/tmp/...`).
- On non-Linux systems, adapt tunnel path/commands accordingly.
